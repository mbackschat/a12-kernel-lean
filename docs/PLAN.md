## Current spiral

The project now follows the controlled-interleaving and risk-first method in [`PROJECT-DESIGN.md`](PROJECT-DESIGN.md#controlled-interleaving-and-coverage-horizons). This is interleaving between completed vertical capsules, not parallel partial implementation, and baseline/risk pairs are a default heuristic rather than a quota. Reuse an existing baseline, keep a selected pair together unless blocked, and reassess frontier order after the pair. Every retained semantic unit closes and commits independently.

1. **Numeric target and application:** over the closed expression outcome, add one nonrepeatable numeric target's checking/rounding and stored rendering, map domain failure to target invalidity with its structured cause, then model placement-sensitive application and prior-target delta. Split target classification from application if the audit exposes more than one new semantic axis. The separate power-fillability risk remains parked: the zero-base/empty-exponent discriminator is known, the required kernel observation is unavailable, and no experimental code remains. Do not repeat it or build a harness; resume only when the observation arrives or a new route is explicitly approved.
2. **Clean-clear versus poison spike:** after target/application exists, hold the stale upstream target and downstream read fixed while varying only the upstream route: existing clean no-selection versus legal rounded division-by-zero invalidity. Both may have a CLEARED-looking delta, but the dependent read must observe ordinary empty in the first case and poison in the second. Do not add a scheduler or general dependency graph.
3. **Evaluator consumer probe:** select a named independent-evaluator task profile in [`IMPLEMENTER-GUIDE.md`](IMPLEMENTER-GUIDE.md), map its required clauses to [`IMPLEMENTATION-MAP.md`](IMPLEMENTATION-MAP.md), and report semantic dependency closure, shipment research closure, and cold-consumer qualification separately. Check whether an isolated reader can state the numeric computation decision procedure, result domain, exclusions, evidence gaps, and separating poison case. Artifact-only readback is the default; an actual Rust/Python candidate, protocol extension, runner, or shipment requires its own adopted capability and any applicable approval.
4. **Whole rule, message, and generated validation:** first assemble one checked whole rule over a supported condition, attaching severity, error field, VALUE/OMISSION type, and an opaque already-resolved message payload without interpolation, partial, or repeatable orchestration. Then, as a separate spike, use one nonrepeatable overlapping-alternatives case to separate computation's first selected operation from its generated validation rule's all-holding mismatch checks.

Then select one syntax-shape legality discriminator. Each is a separate semantic capsule; do not build a general parser, generic lowering framework, publication surface, or custom-oracle runtime.

Before writing Lean for an ambiguous risk, require enough source or retained-observation support to choose the account. If the discriminator is unavailable, record the competing accounts, smallest required observation, and downstream consequence once, add no code, and continue with the next ready frontier. If two consecutive risks park this way, choose one evidence-ready ordinary capsule or propose one batched evidence request; any new harness or governance mechanism still requires an explanation of need, gap, payoff, cost, and alternatives plus explicit approval.

## Guardrails and stop conditions

- Work only in this repository. Sibling repositories remain read-only and visibly unchanged.
- Put verified language-neutral behavior in [`../spec/`](../spec/) and update [`A12-DMKITS-SPEC-SYNC.md`](A12-DMKITS-SPEC-SYNC.md) in the same change. Keep Lean design, implementation status, evidence status, and plans in their `docs/` owners.
- Reuse a completed baseline; never recreate equivalent semantics, lowering, examples, or proofs merely to satisfy the spiral.
- Keep one active capsule or risk spike. A spike must end as a closed capsule, a precise unresolved boundary with experimental code removed, or a representation correction whose affected gates are re-closed. Do not leave parallel half-built semantic branches, and do not generalize a mechanism until a second completed consumer demonstrates the same meaning, result domain, and law.
- After a representation correction, audit the affected theorem statements, hypotheses, directions, result domains, and nearest non-laws rather than accepting compilation alone. Source-grounded clauses may propagate `external evidence pending`; representation-affecting unresolved alternatives block dependent work.
- Use red/green TDD, the Tier 1 gate in [`TESTING.md`](TESTING.md), payoff-selected laws/non-laws, and honest external-evidence status for each capsule.
- Add no dependency and change no linkage, hashing, or distribution policy without explicit approval.
- Do not restore retired V1, raw-packet, generator, registry, qualification, or universal-capture machinery. Any proposed harness or governance work must first be surfaced with the reproduced need, existing-mechanism gap, payoff, cost, and alternatives, then receive explicit user approval.
- Stop only for a serious blocker requiring new authority, an unavailable observation that changes the intended claim, an irreconcilable compatibility choice, or a dependency/licensing decision.
