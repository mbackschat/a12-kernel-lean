# Open semantic gaps

This is the open-only work index for convergence toward complete semantic conformance with A12 kernel 30.8.1. [`IMPLEMENTATION-MAP.md`](IMPLEMENTATION-MAP.md) remains the detailed owner of executable, proof-closed, kernel-calibrated, publicly exposed, and consumer-qualified coverage; [`PLAN.md`](PLAN.md) selects the immediate unit from this index.

## Query contract

Open only the selected `SG<n>` entry. Keystone and cross-family entries use the same readiness labels: `Missing invariant`, `Current boundary`, `Prerequisite owners`, `Unresolved source fact`, `Most dangerous discriminator`, `Consumer hypothesis`, `Forbidden shortcut`, `Evidence status`, `Entry gate`, and `Completion gate`. Semantic-family entries may retain the shorter equivalent when their next unit is already source-closed. Detailed implemented facts stay behind each entry's [`IMPLEMENTATION-MAP.md`](IMPLEMENTATION-MAP.md) links. Search headings with `rg -n '^### SG' docs/SEMANTICS-GAPS.md` and readiness fields with `rg -n '^- (Prerequisite owners|Unresolved source fact|Most dangerous discriminator|Consumer hypothesis|Forbidden shortcut|Evidence status|Entry gate|Completion gate):' docs/SEMANTICS-GAPS.md`. Do not reproduce an SG in [`PLAN.md`](PLAN.md): the plan names the selected capsule and links back here.

## Lifecycle

- A gap exists here only while a kernel behavior, required static legality rule, checked construction, or conformance obligation remains open. Completed gaps are deleted in the completing commit; Git records when they closed, while the resulting capability and evidence status remain in [`IMPLEMENTATION-MAP.md`](IMPLEMENTATION-MAP.md).
- `SG<n>` identifiers are never reused. Before assigning one, search this repository and Git history. A durable conclusion discovered while closing a gap belongs in the relevant [`spec/`](../spec/) clause or, for a non-obvious formalization consequence, [`LEAN-FINDINGS.md`](LEAN-FINDINGS.md).
- Each gap states the readiness fields required by the query contract, with a reopening trigger inside `Unresolved source fact` when blocked or deferred. It does not repeat completed inventories, gate counts, implementation narratives, or immediate scheduling.
- A blocked entry remains here with the exact missing fact or shared boundary and a reopening trigger. A protocol, shipment, dependency, harness, or governance proposal is not a semantic gap and remains with its existing owner and approval rule.

## Hard-frontier critical path

The whole-model path is **SG1 → SG2 → SG4**. SG1 is the sole owner of one immutable model-certified checked document. SG2 consumes that checked document and the existing checked plans to produce one canonical addressed operand stream whose ordered environments retain hierarchical extent, relevance, declaration metadata, and structural failure. SG4 consumes both boundaries with the separate processing context and checked computation plan; it must not redefine document state, addressing, or either checked certificate.

Semantic-family work may continue when it does not freeze one of those future boundaries. SG2 cannot claim general addressing before SG1 supplies the coherent input it addresses, and SG4 cannot begin until SG1 and SG2 have closed their entry and completion gates plus SG4's own bounded source packet.
